User stories

1
Título: Cadastro de usuário

Quem?
	Eu como usuário cadastrado.
O que?
	Devo gerenciar as contas de novos usuários.
Por que?
	Para ter controle sobre os usuários que acessam o sistema.
-Critérios de aceitação
	-A partir do cadastro, o novo usuário está disponível a acessar as funcionalidades.


2
Título: Acesso ao sistema

Quem?
Eu como um usuário cadastrado no sistema.
O que?
	O sistema deve estar disponível tempo todo em qualquer navegador.
Por que?
	Com o objetivo de acessar as funcionalidades do sistema.
-Critérios de aceitação
	-Não permitir que um usuário não cadastrado faça login.
	-Ao realizar o login, devo acessar o sistema.
	-As funcionalidades devem estar disponíveis para qualquer usuário logado.

19
Título: Alteração dos dados de professor

Quem?
	Eu como um usuário cadastrado no sistema.
O que?
	Poder alterar quaisquer dados dos professores cadastrados.
Por que?
	Com objetivo de que os dados do sistema estejam condizentes com a realidade.
-Critérios de aceitação
	-Mensagem de erro ao tentar alterar um campo, utilizando valores que não condizem com o campo.
	-Campo que não podem ser alterados, devido a serem chaves primárias, devem ser desabilitados durante a alteração.

20
Título: Desativar professor

Quem?
	Eu como um usuário cadastrado no sistema.
O que?
	Preciso desativar um professor afastado de uma ou várias atividades.
Por que?
	Para que a informação dele não sejam perdidas após o afastamento do professor, e que sua situação atual no sistema esteja condizente a realidade.
-Critérios de aceitação
	-Desabilitar a desativação de um professor já desativado.
	-Mensagem de erro ao atribuir atividade a um professor desativado.
